## Nix Shell Portability (see feedback_nix-shell-portability.md)
- Nix shell replaces macOS utils with GNU versions (different syntax)
- No `grep -oP`, no `sed -i ''`, no `stat -f`, no `ps`/`lsof`/`launchctl` in PATH
- ALWAYS prefer Claude Code Edit tool over sed. Use R for data extraction.
- Push cachix with ./push_to_cachix.sh (never push deps)
- Check CURRENT_WORK.md + git status + open issues at start

## Never `cd && git` (see feedback_no-compound-cd.md)
- `cd <dir> && git ...` triggers approval prompt that bypassPermissions does NOT bypass
- ALWAYS use `git -C <dir> <subcommand>` instead — every git subcommand accepts -C
- Working example: ~/.claude/hooks/session_stop.sh:39-40
- See `git-no-compound-cd` rule for full substitution table

## Editing Symlinked Config (see feedback_symlink-edit-vs-mv.md)
- ~/.claude/settings.json is a symlink to ~/docs_gh/llm/.claude/settings.json (since 2026-05-05)
- Use Edit tool or shell `>` redirect — `mv source target` REPLACES the symlink with a regular file
- Recovery: `cp current → real_target; rm current; ln -s real_target symlink_path`
