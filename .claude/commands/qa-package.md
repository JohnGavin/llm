# /qa-package Command

Run adversarial QA on the current R package.

## Description

Generates attack vectors and stress tests for all exported functions, identifies defensive programming gaps, and produces a verdict on package robustness.

## Usage

```
/qa-package [options]
```

## Options

| Option | Description | Default |
|--------|-------------|---------|
| `--functions=<list>` | Comma-separated function names to test | All exports |
| `--categories=<list>` | Attack categories to run | All categories |
| `--generate-tests` | Write tests to test-adversarial.R | false |
| `--fix` | Attempt to fix failures | false |

## Examples

```
# Test all exported functions
/qa-package

# Test specific functions
/qa-package --functions=analyze_data,process_metrics

# Run only null and type attacks
/qa-package --categories=null_input,type_coercion

# Generate test file
/qa-package --generate-tests

# Full cycle: test, fix, re-test
/qa-package --fix
```

## Process

### Step 1: Discover Targets

```r
# Find all exported functions
targets <- discover_attack_targets(".")
cli::cli_alert_info("Found {length(targets)} exported functions")
```

### Step 2: Generate Attacks

For each function, generate attack vectors:

- **null_input**: NULL for each argument
- **empty_input**: Empty vectors, data frames, strings
- **type_coercion**: Wrong types for each argument
- **boundary**: 0, -1, Inf, -Inf, NaN for numeric args
- **na_input**: NA, NA_real_, NA_character_
- **data_structure**: tibble vs data.frame, single row, matrix
- **idempotency**: f(f(x)) consistency

### Step 3: Execute Attacks

```bash
nix-shell default.nix --run "Rscript -e 'source(\"R/qa_attacks.R\"); run_attacks()'"
```

### Step 4: Report Results

```markdown

|----------|------|------|------|

```

### Step 5: Generate Tests (if --generate-tests)

```r
# tests/testthat/test-adversarial.R
# Auto-generated by /qa-package

test_that("analyze_data defends against NULL input", {
  expect_error(
    analyze_data(data = NULL),
    class = "rlang_error"
  )
  expect_error(
    analyze_data(data = NULL),
    regexp = "data|NULL|required"
  )
})

test_that("analyze_data handles empty data frame", {
  result <- analyze_data(data = data.frame())
  expect_true(nrow(result) == 0 || inherits(result, "error"))
})

test_that("process_metrics rejects negative n", {
  expect_error(
    process_metrics(n = -1),
    class = "rlang_error"
  )
})
```

## Attack Categories Detail

### null_input
Tests that NULL arguments produce clear error messages.

```r
test_that("function rejects NULL with clear message", {
  err <- expect_error(f(NULL), class = "rlang_error")
  expect_match(conditionMessage(err), "cannot be NULL|required")
})
```

### type_coercion
Tests that wrong types produce errors mentioning the expected type.

```r
test_that("function rejects wrong type", {
  err <- expect_error(f("string"))  # When numeric expected
  expect_match(conditionMessage(err), "numeric|character")
})
```

### boundary
Tests numeric edge cases: 0, negative, Inf, NaN.

```r
test_that("function handles boundary values", {
  expect_error(f(Inf))
  expect_error(f(NaN))
  expect_error(f(-1))  # If positive required
})
```

### na_input
Tests NA handling across types.

```r
test_that("function handles NA explicitly", {
  expect_error(f(NA), regexp = "NA|missing")
  # OR
  result <- f(c(1, NA, 3))
  expect_true(!anyNA(result) || attr(result, "na_handled"))
})
```

### data_structure
Tests tibble vs data.frame, single row, matrix input.

```r
test_that("function works with tibble and data.frame", {
  df_result <- f(data.frame(x = 1:3))
  tbl_result <- f(tibble::tibble(x = 1:3))
  expect_equal(df_result, tbl_result)
})
```

## Integration

### With Quality Gates
Pass rate contributes to defensive programming score.

### With Verifier
Verifier can run `/qa-package` as part of pre-PR checks.

### With Orchestrator
Orchestrator can trigger `/qa-package --fix` in the FIX phase.

## Model Usage

| Phase | Model | Rationale |
|-------|-------|-----------|
| Discover functions | haiku | Simple parsing |
| Generate attacks | haiku | Template-based |
| Run attacks | sonnet | Parse test output |
| Suggest fixes | sonnet | Code generation |
| Write test file | sonnet | Code generation |
