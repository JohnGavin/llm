# Snapshot Conversion Examples

Five worked before/after pairs for migrating `expect_equal(actual, "string")` to
`expect_snapshot()`. Each shows why snapshot tests improve review experience and
maintenance ergonomics.

---

## 1. Simple string equality

**Before**

```r
test_that("format_status returns correct string", {
  x <- format_status("pending")
  expect_equal(x, "Status: pending (0%)")
})
```

**After**

```r
test_that("format_status returns correct string", {
  x <- format_status("pending")
  expect_snapshot(x)
})
```

**Why better:** When the format changes (e.g. richer emoji prefix), the before-test
fails with a cryptic character-level diff. The snapshot test shows the full old vs
new string side by side in `_snaps/test-status.md`, making the intent of the change
obvious in code review. Accepting the new format is `snapshot_accept("status")`.

## 2. Multi-line string

**Before**

```r
test_that("render_report produces expected text", {
  out <- render_report(list(n = 3, status = "ok"))
  expect_equal(out, "Report\n  n: 3\n  status: ok\n")
})
```

**After**

```r
test_that("render_report produces expected text", {
  out <- render_report(list(n = 3, status = "ok"))
  expect_snapshot(cat(out))
})
```

**Why better:** The escaped `\n` in the before-test is unreadable. The snapshot file
stores the rendered text with actual line breaks, making it easy to eyeball the layout.
`cat()` wraps the value so the snapshot shows printed output rather than `[1] "..."`.

## 3. List with strings

**Before**

```r
test_that("parse_result returns named list", {
  x <- parse_result("copied:mypkg:0.2.0")
  expect_equal(x$action, "copied")
  expect_equal(x$package, "mypkg")
  expect_equal(x$version, "0.2.0")
})
```

**After**

```r
test_that("parse_result returns named list", {
  x <- parse_result("copied:mypkg:0.2.0")
  expect_snapshot(x)
})
```

**Why better:** Three separate `expect_equal` calls each assert one field, but the
snapshot test captures the entire structure in one step. If a new field is added
legitimately, the snapshot diff flags it for review — three separate assertions would
silently miss the extra field.

## 4. Formatted print output

**Before**

```r
test_that("print.review_summary shows correct summary", {
  obj <- new_review_summary(n_open = 2, n_closed = 5)
  expect_equal(format(obj), "Reviews: 2 open, 5 closed (71% closed)")
})
```

**After**

```r
test_that("print.review_summary shows correct summary", {
  obj <- new_review_summary(n_open = 2, n_closed = 5)
  expect_snapshot(print(obj))
})
```

**Why better:** `format()` returns the string silently; `print()` fires the S3 method
under test. `expect_snapshot_output(print(obj))` is equivalent and also valid.
Using `print()` ensures the full display contract — including any header/footer lines
the `print` method adds — is under test. Hardcoding `format()` output misses those.

## 5. File contents

**Before**

```r
test_that("write_report creates expected file", {
  tmp <- withr::local_tempfile(fileext = ".txt")
  write_report(list(status = "done"), tmp)
  expect_equal(readLines(tmp), c("Status: done", "---", "Generated by llm"))
})
```

**After**

```r
test_that("write_report creates expected file", {
  tmp <- withr::local_tempfile(fileext = ".txt")
  write_report(list(status = "done"), tmp)
  expect_snapshot_file(tmp, "report.txt")
})
```

**Why better:** `expect_snapshot_file()` stores a copy of the file in
`tests/testthat/_snaps/test-report/report.txt`. The snapshot is committed to git and
diffs are visible in PR review as plain text. The before-test requires every
contributor to manually update the hard-coded vector when the footer changes; the
snapshot approach makes that one command: `snapshot_accept("report.txt")`.

## Quick-reference decision table

| Test pattern | Snapshot approach |
|---|---|
| `expect_equal(x, "some string")` | `expect_snapshot(x)` |
| `expect_equal(x, "line1\nline2\n")` | `expect_snapshot(cat(x))` |
| Multiple `expect_equal(x$a, "foo"); expect_equal(x$b, "bar")` | `expect_snapshot(x)` |
| `expect_equal(format(x), "...")` | `expect_snapshot(print(x))` or `expect_snapshot_output(print(x))` |
| `expect_equal(readLines(f), c(...))` | `expect_snapshot_file(f, "name.ext")` |
