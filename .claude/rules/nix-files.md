# Nix Environment Rules

**Applies to**: `*.nix`, `default.R`, `default.sh`

## The Iron Rule: NEVER Install Inside Nix

```r
# FORBIDDEN - Breaks Nix immutability
install.packages("dplyr")        # NO!
devtools::install()              # NO!
pak::pkg_install("tidyverse")    # NO!
remotes::install_github(...)     # NO!

# ALLOWED - Safe operations
devtools::load_all()     # Temporary load for development
devtools::document()     # Updates docs
devtools::test()         # Runs tests
devtools::check()        # Full R CMD check
```

**Why**: Nix guarantees reproducibility. Installing packages breaks this guarantee.

## Adding New Dependencies

When a package needs a new dependency:

1. **Edit DESCRIPTION**: Add to `Imports:` or `Suggests:`
2. **Regenerate default.nix**: Run `source("default.R")`
3. **Exit Nix shell**: Exit current shell
4. **Re-enter**: Run `./default.sh`

```r
# In default.R - regenerates default.nix from DESCRIPTION
rix::rix_from_description(
  project_path = ".",
  r_ver = "4.5.0",
  ide = "other",
  overwrite = TRUE
)
```

## default.sh Structure

The standard `default.sh` with GC root:

```bash
#!/usr/bin/env bash
set -e

# Build the Nix expression and create GC root
NIX_STORE_PATH=$(nix-build default.nix --no-out-link)

# Create/update GC root to prevent garbage collection
nix-store --add-root ./nix-shell-root --indirect -r "$NIX_STORE_PATH"

# Enter the shell
nix-shell default.nix
```

**Key features**:
- `set -e`: Exit on any error
- `--no-out-link`: Don't create result symlink
- `--add-root`: Creates GC root to persist packages
- `./nix-shell-root`: Symlink preventing garbage collection

## GC Root Management

```bash
# First run: Builds everything (slow)
./default.sh

# Subsequent runs: Uses cached packages (fast)
./default.sh

# Force rebuild (when needed)
rm nix-shell-root && ./default.sh
```

## Files to Exclude (CRITICAL)

**Add to .gitignore**:
```
nix-shell-root
```

**Add to .Rbuildignore**:
```
^nix-shell-root$
^default\.R$
^default\.nix$
^default\.sh$
^default-ci\.nix$
```

**Why**: `nix-shell-root` is a symlink to `/nix/store/` which doesn't exist on other machines.

## default.nix Structure

Generated by rix, but key elements:

```nix
{ pkgs ? import (fetchTarball {
    url = "https://github.com/NixOS/nixpkgs/archive/...";
    sha256 = "...";
  }) {}
}:

pkgs.mkShell {
  buildInputs = with pkgs; [
    R
    rPackages.dplyr
    rPackages.tidyr
    # ... other packages
  ];

  shellHook = ''
    export R_LIBS_USER=""
    export IN_NIX_SHELL=1
  '';
}
```

## Cachix Integration

For faster CI and team builds:

```nix
# In default.nix or flake.nix
substituters = [
  "https://cache.nixos.org"
  "https://rstats-on-nix.cachix.org"
  "https://johngavin.cachix.org"  # Your personal cache
];
```

**Push to cache after building**:
```bash
# After successful build
nix-build default.nix | cachix push johngavin
```

## CI Considerations

For GitHub Actions with Nix:

1. **Use nix-shell --run**: Not interactive shells
2. **Cache /nix/store**: Speed up CI runs
3. **Don't expect default.sh**: CI doesn't have persistent shells

```yaml
# .github/workflows/check.yml
- name: Run tests
  run: |
    nix-shell default.nix --run "Rscript -e 'devtools::test()'"
```

## Two-Tier Architecture

Claude operates in two Nix shells:

| Shell | Purpose | Has Project Packages? |
|-------|---------|----------------------|
| Development | Generic R dev tools | No |
| Project-specific | From DESCRIPTION | Yes |

**Agents must use**:
```bash
nix-shell default.nix --run "Rscript -e 'command_here'"
```

## Debugging Nix Issues

```bash
# Verify you're in Nix
echo $IN_NIX_SHELL  # Should be 1

# Check R location
which R  # Should be /nix/store/...

# Check available packages
R -e "installed.packages()[,'Package']"

# If packages missing, regenerate
source("default.R")
exit  # Exit current shell
./default.sh  # Re-enter
```
